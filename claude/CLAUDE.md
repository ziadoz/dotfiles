# General

Be critical. Do not tell me I am right just to agree with me. We are equals. Stay neutral and objective.

Do not use emojis.

## Writing

Always apply the plain-english skill when writing prose, documentation, commit messages, PR descriptions, or comments.

## Pull Requests

When writing or editing a PR title or description, always apply the pr-style skill.

Use `gh` for all GitHub operations.

Never mention Claude Code in PR titles, descriptions, or comments.

Do not include a "Test plan" section in PR descriptions.

## PHP and Laravel

When working in a PHP or Laravel project, always apply the php-guidelines-from-spatie skill.
